# v0.2.0 Release Checklist

This checklist prepares the repository for a future `v0.2.0` tag.
Completing it does not create a tag or GitHub Release.

## Package and Static Validation

- [ ] Install the package from a clean environment with `python -m pip install -e ".[dev]"`.
- [ ] Confirm `ase --version` prints `0.2.0` from package metadata.
- [ ] Run `python -m pytest -q` with no failures.
- [ ] Run `python -m mypy agent_sandbox_eval` with no issues.
- [ ] Run `ase validate-tasks --benchmark all` for all bundled manifests.
- [ ] Run `ase lint-tasks --benchmark all --probe-setup` with Docker available.

## Docker Controls

- [ ] Run the full scripted oracle across all bundled tasks.
- [ ] Run the full ReAct local-solution control across all bundled tasks.
- [ ] Run the full noop negative control across all bundled tasks.
- [ ] Confirm the oracle and local-solution controls pass every task.
- [ ] Confirm the noop control passes no tasks.
- [ ] Confirm all controls contain zero real model calls.

## Matrix Evidence

- [ ] Run `ase run-matrix examples/experiments/local-controls.yaml`.
- [ ] Run the same command again and confirm every unit resumes with zero executed attempts.
- [ ] Confirm the regenerated `summary.json` and `report.md` are byte-stable across the resume.
- [ ] Run `python -m pytest -q tests/test_matrix_runner.py` to prove corruption fails closed.
- [ ] Confirm the reports separate first attempts, retries, environment failures, and budget failures.
- [ ] Confirm estimated cost is unavailable for unpriced controls.

## Publication Boundary

- [ ] Keep live-model result fields empty unless an operator authorizes and funds a real-model run.
- [ ] Record an explicit pricing source before publishing any estimated cost.
- [ ] Describe scripted, local-solution, and noop results as harness controls rather than capability claims.
- [ ] Review the pull request and all required checks before a maintainer creates any tag.
